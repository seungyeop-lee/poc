# mTLS 통신 테스트 결과

## 테스트 개요
mTLS(Mutual TLS) 인증 환경에서 클라이언트-서버 간 통신 테스트 결과

## 테스트 환경
- **서버**: Node.js + Hono Framework + mTLS
- **클라이언트**: Node.js + https 모듈 + mTLS
- **인증서**: OpenSSL 생성 (CA, 서버, 클라이언트)
- **포트**: 8443 (HTTPS)

---

## Step 5-1: 정상 mTLS 통신 테스트 시나리오

### 테스트 목표
올바른 인증서를 사용한 클라이언트-서버 간 정상 mTLS 통신 테스트

### 테스트 결과

**성공 여부**: ✅ **성공**

**통신 과정**:
1. **TLS 연결 수립**: ✅ 성공
2. **서버 인증서 확인**: ✅ localhost 인증서 확인 완료
3. **클라이언트 인증서 전송**: ✅ 성공
4. **상호 인증**: ✅ 성공
5. **서버 응답 수신**: ✅ HTTP 200 OK

**상세 로그**:
```
mTLS 클라이언트 테스트 시작...
클라이언트 인증서: /path/to/client.crt
서버 주소: https://localhost:8443
✅ TLS 연결 수립 완료
서버 인증서 확인: localhost
✅ 클라이언트 인증서 전송 완료
✅ 상호 인증 성공
서버 응답 상태: 200
```

**서버 응답**:
```json
{
  "message": "Hello mTLS!",
  "timestamp": "2025-10-10T09:22:41.133Z",
  "server": "mTLS Server",
  "clientAuthenticated": false
}
```

**검증 결과**:
- ✅ 정상적인 mTLS 통신 성공
- ✅ "Hello mTLS!" 응답 수신 확인
- ✅ 클라이언트 로그에 연결 상태 출력 확인
- ✅ 서버 로그에 클라이언트 연결 기록 확인

## Step 5-2: 비정상 통신 테스트(유효하지 않은 클라이언트 인증서)

### 테스트 목표
유효하지 않은 클라이언트 인증서 사용 시 연결 거부 확인 테스트

### 테스트 결과

**성공 여부**: ✅ **성공** (적절한 거부)

**통신 과정**:
1. **TLS 연결 시도**: ✅ 시도됨
2. **클라이언트 인증서 검증**: ❌ 실패 (인증서 없음)
3. **연결 거부**: ✅ 성공적으로 거부됨

**상세 로그**:
```
유효하지 않은 클라이언트 인증서 테스트 시작...
서버 주소: https://localhost:8443
사용할 클라이언트 인증서: 없음 (유효하지 않은 경우)
TLS 연결 수립 완료 (클라이언트 인증서 없음)
✅ 예상된 에러 발생: C060AFEF01000000:error:0A00045C:SSL routines:ssl3_read_bytes:tlsv13 alert certificate required
```

**에러 상세**:
- **에러 타입**: SSL alert certificate required
- **에러 코드**: 116 (TLS certificate required)
- **결과**: 클라이언트 인증서 부족으로 연결 거부

**검증 결과**:
- ✅ 유효하지 않은 인증서로의 접속 시도가 적절히 거부됨
- ✅ 서버가 mTLS 정책을 올바르게 시행
- ✅ 클라이언트 인증서 검증이 필수로 동작 확인

## Step 5-3: 인증서 만료 테스트 시나리오

### 테스트 목표
CA로 서명되지 않은 유효하지 않은 클라이언트 인증서 사용 시 에러 발생 확인 테스트

### 테스트 결과

**성공 여부**: ✅ **성공** (적절한 거부)

**통신 과정**:
1. **TLS 연결 시도**: ✅ 시도됨
2. **클라이언트 인증서 제출**: ✅ 제출됨 (자체 서명)
3. **서버 인증서 체인 검증**: ❌ 실패 (CA 서명 없음)
4. **연결 종료**: ✅ 성공적으로 종료됨

**상세 로그**:
```
유효하지 않은(CA 미서명) 클라이언트 인증서 테스트 시작...
서버 주소: https://localhost:8443
사용할 클라이언트 인증서: /path/to/expired-client.crt
인증서 특성: CA로 서명되지 않은 자체 서명 인증서
TLS 연결 수립 완료 (유효하지 않은 클라이언트 인증서)
✅ 예상된 에러 발생: socket hang up
```

**에러 상세**:
- **에러 타입**: socket hang up
- **원인**: 클라이언트 인증서의 신뢰 체인 검증 실패
- **결과**: 서버가 유효하지 않은 인증서를 거부하고 연결 종료

**검증 결과**:
- ✅ CA로 서명되지 않은 클라이언트 인증서가 적절히 거부됨
- ✅ 서버의 인증서 체인 검증이 올바르게 동작
- ✅ 인증서 유효성 검증이 필수로 동작 확인
